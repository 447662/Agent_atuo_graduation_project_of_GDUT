# 毕业论文 Markdown 转 LaTeX

把已经写好并经用户确认的毕业论文 `.md` 转换为基于 LaTeX 模板的 `.tex` 文件。该 skill 不负责重新写论文正文，重点是确认模板、转换正文结构、保留模板工程约束并交付可继续编辑的 `.tex`。

## 模板规则

转换前必须先询问用户是否需要上传或指定新的 LaTeX 模板。

- 如果用户指定新的模板文件，优先使用用户提供的模板。
- 如果用户只提供模板目录，要求用户确认主入口 `.tex` 文件。
- 如果用户明确不需要新模板，使用默认模板：`/g:/My_code/Agent_atuo_graduation_project_of_GDUT/模板/GDUT-Undergraduate-Thesis-LaTeX-Template-main/模板.tex`。
- 如果模板文件缺失、不可读或入口不明确，先暂停并要求用户补充，不要假设模板结构。

默认模板可能依赖同目录下的 `.cls`、`.sty`、图片、参考文献样式或其他资源；转换时应保留模板工程的相对路径关系，不要把正文输出成与模板资源脱离的孤立文件。

## 启动流程

1. 确认输入 Markdown 文件路径。
   - 如果用户提供单个 `.md` 文件，转换该文件。
   - 如果用户提供目录，询问是转换目录下所有顶层 `.md`，还是只转换某一个文件。
2. 询问用户是否需要上传或指定新的 LaTeX 模板。
3. 根据用户回答确定模板：
   - 用户提供模板文件：使用该文件。
   - 用户提供模板目录：要求确认主入口 `.tex`。
   - 用户不需要新模板：使用默认模板路径。
4. 检查模板文件是否存在，并确认它是可作为入口或正文承载模板使用的 `.tex` 文件。
5. 将 Markdown 正文转换为 LaTeX 内容，并按模板结构整合：
   - 若模板有明确正文占位或 `\input{}` 结构，优先把正文放入对应位置或章节文件。
   - 若模板是单文件入口，保留导言区、文档类、宏包和封面配置，只替换或插入正文区域。
   - 若无法判断正文插入点，先询问用户，不要破坏模板。
6. 转换后检查明显问题：图片路径、公式、表格、参考文献、特殊字符、标题编号、未闭合环境和模板资源引用。
7. 输出最终 `.tex` 路径，并简要说明使用了哪个 Markdown、哪个 LaTeX 模板和哪些内容需要人工复查。

## Markdown 预处理要求

转换前检查并必要时提醒用户修正：

- 标题层级应清晰，Markdown 中可使用 `# 1 绪论`、`## 1.1 研究背景` 等带编号标题帮助识别层级；转换为 LaTeX 时要去掉标题文本中的编号，让 LaTeX 自动编号，例如输出 `\section{绪论}`、`\subsection{研究背景}`，不要输出 `\section{1 绪论}` 或 `\subsection{1.1 研究背景}`。
- 摘要、Abstract、目录、正文、结论、参考文献、致谢、附录应结构完整；生成 LaTeX 时摘要标题应居中显示，中文摘要与英文 Abstract 都要按模板的居中标题样式处理。
- 转换为 LaTeX 章节结构时，每个大章节应独立另起一页；在章级标题、摘要、Abstract、目录、参考文献、致谢、附录等主要部分前使用模板推荐的分页方式，必要时插入 `\clearpage` 或 `\newpage`。
- 图表应有规范题注：`图X.Y 图名`、`表X.Y 表名`。
- 图片路径应相对 Markdown 文件、项目根目录或模板工程可访问。
- 数学公式应使用 Pandoc/LaTeX 可识别的 `$...$`、`$$...$$` 或 LaTeX 环境。
- 参考文献不要使用虚构信息；若模板使用 BibTeX/BibLaTeX，应说明需要用户确认 `.bib` 或文献条目映射。
- LaTeX 特殊字符如 `%`、`_`、`&`、`#`、`{}` 可能需要转义，避免破坏编译。
- Mermaid、HTML、Obsidian 特殊语法、复杂表格和代码块可能无法 1:1 映射，应列为人工复查项。

## 输出文件规则

- 默认输出到源 Markdown 同目录。
- 默认文件名与源 Markdown 同名，例如 `thesis.md` → `thesis.tex`。
- 如果目标文件已存在，不要无提示覆盖；使用带后缀的新文件名，或询问用户是否覆盖。
- 转换成功后只报告必要信息：源 Markdown、输出 TEX、使用模板、是否存在需要人工复查的项。

## 常见问题处理

- **模板缺失**：提示用户补充 LaTeX 模板文件，或确认是否改用默认模板。
- **模板入口不明确**：要求用户指定主入口 `.tex`，不要自行猜测。
- **图片丢失**：检查 Markdown 所在目录、项目根目录、模板工程目录和 LaTeX `\includegraphics` 路径。
- **公式或特殊字符异常**：说明需要转义或人工复查，必要时先修正 Markdown 后再转换。
- **表格过宽或复杂表格失败**：尽量转换为 LaTeX 表格，并列为人工复查项。
- **参考文献格式异常**：说明当前输出是否为纯文本参考文献，或是否需要用户提供 `.bib` 文件。
- **用户要求直接生成最终 PDF**：可以在 `.tex` 生成后说明是否具备编译条件；缺少 TeX 环境或模板依赖时不要承诺能编译通过。
