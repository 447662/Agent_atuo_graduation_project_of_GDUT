# 文献翻译转 LaTeX

把项目 `文献` 文件夹下用户选择的外文文献整理为基于 GDUT LaTeX 模板的文献翻译 `.tex` 文档。该 skill 负责选择文献、提取原文、按自然段翻译、套用固定目录结构、替换封面配置，并把最终文件生成在 `文献` 文件夹下。

## 固定路径

- 文献目录：项目根目录下的 `文献/`。
- 默认 LaTeX 模板：`/g:/My_code/Agent_atuo_graduation_project_of_GDUT/模板/GDUT-Undergraduate-Thesis-LaTeX-Template-main/模板.tex`。
- 文献翻译封面：`/g:/My_code/Agent_atuo_graduation_project_of_GDUT/模板/文献翻译封面.pdf`。
- 默认输出文件：`文献/文献翻译.tex`。

如果用户明确指定其他文献目录、模板或输出文件名，以用户指定为准；否则使用以上默认值。

## 触发后的启动流程

1. 定位项目根目录下的 `文献/` 文件夹。
2. 列出可用文献文件，让用户选择要翻译的文献；如果用户已经明确指定文件，直接使用该文件。
3. 确认输入文件类型并提取正文：
   - PDF：使用 PDF 读取/OCR 能力提取文本；扫描版或无法复制的 PDF 需要说明 OCR 可能有误。
   - 加密或权限受限 PDF：先尝试读取元数据；若正文无法提取，暂停并要求用户提供未加密版本、可复制文本、DOCX/TXT 版本或密码，不要编造译文和文献信息。
   - DOCX：使用 Word 文档读取能力提取段落。
   - Markdown/TXT/LaTeX：直接读取文本内容。
   - CAJ/其他不可直接读取格式：先提示需要转换为 PDF、DOCX、TXT 或提供可提取文本版本。
4. 读取默认 LaTeX 模板，保留模板导言区、宏包、版式配置和文档结构。
5. 将模板中的封面命令替换为文献翻译封面。
6. 按规定目录结构生成文献翻译内容。
7. 写入 `文献/文献翻译.tex`；若该文件已存在，不要无提示覆盖，先询问用户是否覆盖，或生成带后缀的新文件名。
8. 简要报告源文献、输出路径、是否有 OCR/格式/参考信息需要人工复查。

## 封面替换规则

默认模板里可能存在类似配置：

```latex
% 封面设置
% 封面设置（仅当封面文件存在时插入，避免编译因找不到文件失败）
\newcommand{\makecover}{%
      \IfFileExists{论文封面.pdf}{\includepdf[pages=-]{论文封面.pdf}}{}%
}
```

生成文献翻译 `.tex` 时，应将封面文件改为模板目录中的 `文献翻译封面.pdf`。优先使用相对路径，确保编译入口所在位置能找到该 PDF；如果输出文件在 `文献/` 下且直接编译该文件更方便，可使用可解析的相对路径或绝对路径。

推荐替换为：

```latex
% 封面设置
% 封面设置（仅当封面文件存在时插入，避免编译因找不到文件失败）
\newcommand{\makecover}{%
      \IfFileExists{/g:/My_code/Agent_atuo_graduation_project_of_GDUT/模板/文献翻译封面.pdf}{\includepdf[pages=-]{/g:/My_code/Agent_atuo_graduation_project_of_GDUT/模板/文献翻译封面.pdf}}{}%
}
```

如果当前 TeX 环境不支持 `/g:/...` 形式的绝对路径，则改用从输出文件到封面 PDF 的相对路径，并在完成说明中提示用户确认封面路径。

## 文档目录结构

最终正文必须使用以下一级和二级结构：

```text
1 外文文献译文（1）
1.1 译文信息
1.2 翻译正文
2 外文文献原文（1）
2.1 文献原文
```

LaTeX 中通常映射为：

```latex
\section{外文文献译文（1）}
\subsection{译文信息}
...
\subsection{翻译正文}
...
\section{外文文献原文（1）}
\subsection{文献原文}
...
```

不要把编号写进标题文本中，除非模板没有自动编号能力。优先让 LaTeX 自动编号。

## 译文信息

`1.1 译文信息` 应尽量包含可从文献中提取到的信息：

- 外文题名
- 中文译名
- 作者
- 来源/期刊/会议/出版社
- 年份
- DOI 或 URL（如有）

缺失信息不要编造；用“未在原文中明确给出”或省略该项，并在完成说明中列为需复查项。

## 翻译正文规则

- 按自然段翻译，不要按句子碎片化翻译。
- 每次处理一个自然段；如果原文自然段过长，应按语义边界拆分，单次翻译内容不超过 500 个中文字或约 500 个英文单词对应的内容。
- 保留原文的技术术语、变量名、算法名、模块名、公式编号、图表编号和引用编号。
- 专业术语第一次出现时可采用“中文译名（英文原词）”，后文保持译名一致。
- 不要扩写、改写或添加原文没有的信息；翻译以忠实、通顺、学术化为目标。
- 公式、代码、表格、图题、图注、参考文献编号应尽量保留原样，必要时转换为 LaTeX 可编译格式。
- 如果原文含图表但无法直接嵌入图片，可保留图题/图注并说明图片需人工补齐。

## 原文保留规则

`2.1 文献原文` 中应放入被翻译的外文原文。

- 保留原始段落顺序。
- 对 LaTeX 特殊字符进行转义，避免 `%`、`_`、`&`、`#`、`{}` 等破坏编译。
- 原文中的公式、引用编号、图表编号尽量保留。
- 如果输入来源是 PDF/OCR，应说明原文文本可能存在识别错误。

## 加密或无法提取文本的文献

如果 PDF 加密、受权限限制、扫描质量过差或工具无法提取正文：

- 不要继续生成看似完整的译文；这会造成虚构内容风险。
- 可以生成一个流程测试或占位文件，但必须明确标注“未能提取原文，不能作为正式译文”。
- 正式任务中应暂停并请求用户提供未加密 PDF、密码、可复制文本、DOCX/TXT 版本，或允许改用 OCR。
- 只能使用文件名、PDF 元数据或用户明确提供的信息填写译文信息；无法确认的作者、来源、年份、DOI 不要猜测。

## LaTeX 整合要求

- 保留模板的文档类、宏包、页眉页脚、目录、封面命令和编译配置。
- 仅替换或插入正文区域；不要随意删除模板依赖资源。
- 如果模板包含摘要、致谢、参考文献等毕业论文专用部分，而文献翻译不需要，应优先移除或注释这些与文献翻译无关的正文内容，同时保留必要导言区。
- 中文内容应使用模板已配置的中文支持，不要额外引入冲突宏包。
- 生成后检查：标题层级、封面路径、特殊字符、长表格、公式、图片路径、未闭合环境。

## 文件命名和覆盖

默认输出为 `文献/文献翻译.tex`。

- 如果文件不存在，直接写入。
- 如果文件已存在，先询问用户是否覆盖。
- 如果用户不想覆盖，使用 `文献翻译_1.tex`、`文献翻译_2.tex` 等递增命名。

## 完成回复格式

转换完成后，用简短清单说明：

```text
文献翻译 LaTeX 已生成：
- 源文献：...
- 输出文件：文献/文献翻译.tex
- 使用模板：...
- 使用封面：...
- 需人工复查：...
```

若未能完成，说明卡在哪一步以及用户需要补充什么文件或确认什么选择。
